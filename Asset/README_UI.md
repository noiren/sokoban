# GBA Sokoban - UI/デザインチーム向けワークフローガイド

UIデザイナーおよびアセット制作者の皆様へ。
本プロジェクトでは、プログラムに触れることなく、安全かつ簡単に画面レイアウトや素材の追加ができるようにディレクトリ構造が整理されています。

## 📁 担当ディレクトリについて

皆様が作業を行うのは、原則として **`Asset/`** フォルダ内のみとなります。

*   **`Asset/graphics/`**: 画像素材を配置する場所です。
    *   `stills/`: 背景画像（240x160の `.bmp`）や、大きな一枚絵を配置します。
    *   `sprites/`: UIパーツやキャラクターなどのスプライト素材（`.bmp`）を配置します。
    *   `csv/`: パーツの仕様書など（`assets_specification_list.csv`）が含まれています。
*   **`Asset/audio/`**: BGMやSEの音声素材ファイル（`.wav`, `.mod`, `.it`など）を配置します。
    *   マニフェスト **`audio_manifest.json`**（version 2）で ID とファイルを対応付けます。SE は **`se_categories`** と各行の **`category` / `number`**（カテゴリ内で一意）を設定します。
    *   編集は `Asset/tools/audio_bgm_editor/run_audio_bgm_editor.bat` と `Asset/tools/audio_se_editor/run_audio_se_editor.bat`（要 Python）から行えます。行の **▶** でプレビュー、**■** で停止（別行の **▶** で再生切り替え時も前の再生は止まります）。PC では **ffmpeg の `ffplay` が PATH にあること**が必要。**保存**時に検証します。未保存で閉じるときは確認ダイアログが出ます。
    *   ビルド前処理の **`src/tools/ui_compiler.py`** が、`Asset/layouts/*.json` から UI 用 `.h` を生成した直後に **`generate_audio_assets`** を実行し、`src/game/include/generated/audio_ids.h` と `audio_dispatch_*.gen.h` を更新します（`build.bat` / `make` 経由で自動）。単体だけ動かすときは `python src/tools/generate_audio_assets.py`。
*   **`Asset/layouts/`**: UIエディタで作成した**画面の配置データ（`.json`）の最終的な保存・納品先**です。
*   **`Asset/tools/ui_editor/`**: 画面レイアウトを作成するための「UIエディタ」が入っています。

---

## 📦 UIチームの納品物と格納場所について

UIチームからプログラマーへ納品していただく（＝ファイルを保存していただく）データは以下の3種類です。

1. **背景・一枚絵素材 (stills)**
   * **形式**: `.bmp` (240x160ピクセル, 256色または16色インデックス)
   * **格納先**: `Asset/graphics/stills/` (必要に応じてサブフォルダも可)
2. **UIパーツ・キャラ素材 (sprites)**
   * **形式**: `.bmp` (各パーツごとのサイズ, 256色または16色インデックス。幅と高さは必ず8の倍数にしてください)
   * **格納先**: `Asset/graphics/sprites/` (必ず用途別のサブフォルダに入れてください)
3. **画面の配置データ (レイアウト)**
   * **形式**: `.json`
   * **格納先**: **`Asset/layouts/`**
   * **作成方法**: 手書きではなく、後述の「UIエディタ」を使って画面をポチポチ組み上げ、`Ctrl + S` で保存すれば**自動的にこのフォルダに出力（納品）されます。**

## 🎨 画面レイアウトの作成・編集フロー

### 1. ツールを起動する
1. `Asset/tools/ui_editor/` フォルダを開きます。
2. **`run_ui_editor.bat`** をダブルクリックして起動します。
3. エディタの画面が立ち上がります。（※Pythonがインストールされている必要があります）

### 2. 画面をデザインする
*   **背景の選択**: 右側の「Background:」プルダウンから、`Asset/graphics/stills/` に入っている背景画像を選択できます。
*   **パーツの追加**: 「Add Sprite」ボタンを押し、`Asset/graphics/sprites/` 以下の各種パーツを選択して画面に配置します。
*   **パーツの移動**: 配置したパーツはマウスでドラッグ＆ドロップして自由に動かせます。
*   **グリッドスナップ**: 「Snap to Grid (8x8)」にチェックを入れると、GBAの仕様に合わせた8ピクセル単位での正確な配置が可能です。

### 3. データを保存する
*   作業が終わったら、キーボードの **`Ctrl + S`** を押すか、画面下部の「Save JSON」ボタンを押します。
*   ファイル名を付けて保存すると、自動的に **`Asset/layouts/`** フォルダに保存されます。
*   保存された `.json` ファイルは、次回のゲームビルド時に自動的にゲーム内に組み込まれます。

---

## 🖼️ 新しい画像素材を追加したい場合

1.  Aseprite等のツールで画像を作成し、`.bmp` 形式で書き出します。
    *   ※GBAの仕様上、インデックスカラー（256色または16色パレット）である必要があります。
2.  用途に合わせて `Asset/graphics/` 以下の適切なフォルダ（背景なら `stills/`、パーツなら `sprites/` のサブフォルダ）に `.bmp` ファイルを格納してください。
    *   **重要：`.json` ファイルを手動で作成する必要はありません。** ビルド時に自動生成されます。
3.  UIエディタを再起動すると、新しく追加した画像がプルダウンや選択リストに表示されるようになります。

## ❓ よくある質問

*   **Q. プログラムが壊れないか心配です。**
    *   A. `Asset/` フォルダ内での作業であれば、ソースコードが壊れることはありません。自由に変更・保存を行ってください。
*   **Q. 何のパーツがどこにあるか分かりません。**
    *   A. `Asset/graphics/csv/assets_specification_list.csv` に、各パーツのファイル名と用途のリストが記載されていますので参考にしてください。
