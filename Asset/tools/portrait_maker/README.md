# GBA アセットメーカー（立ち絵 / 汎用焼き）

起動すると **タブが2つ** あります。

1. **立ち絵** — キャラクターの「感情ごとの立ち絵」を PNG から GBA 用 BMP に変換します（従来の Portrait Maker）。
2. **汎用焼き** — 任意の画像を **スプライト用（16色・index0透過）** または **スチル背景用（256色・index0予約）** に変換します。メニュー紙・ギャラリー部品・イベント背景など、用途に合わせてサイズとモードを選んでください。

プログラムの知識がなくても使えます。

## 起動
`run_portrait_maker.bat` をダブルクリックします。

## UI エディタでの検証

レイアウト JSON に載せた **bg / sprite** が指す BMP について、色数・サイズ・index0 の扱いなどをまとめて確認するには、**GBA UI Editor** のツールバー **「素材検証(BMP)」** を使います（`gba_asset_bake` と同じルール）。

## 立ち絵タブの使い方（4ステップ）

1. **キャラクターを選ぶ**
   - 画面上の「キャラクター」から選びます。新しいキャラは「キャラ追加」で作れます。

2. **PNGを用意する**
   - 「フォルダを開く」を押すと、そのキャラ専用の画像フォルダが開きます。
   - 透過PNG（背景が透明な立ち絵）をそこに入れておくと選びやすいです。

3. **感情にPNGを割り当てる**
   - 中央の「感情スロット」から感情を選び、「PNGを割り当て」で画像を選びます。
   - 右の「切り出し調整」で、
     - **ドラッグ**＝表示位置の移動
     - **マウスホイール / ズームスライダー**＝拡大・縮小
     - **自動(顔)**＝顔まわりに自動で合わせる
   - 右下に「実機での見た目（64×64）」が表示されます。

4. **変換する**
   - 「GBA素材に変換(すべて)」を押すと、割り当てたPNGがまとめて素材(BMP)になります。
   - その後、ゲームをビルド（`build.bat` など）すると反映されます。

## 汎用焼きタブ

1. **参照…** で PNG などを開く。
2. **出力種別** で「スプライト」か「スチル背景」を選ぶ（ルールは `Asset/ASSET_GUIDELINES.md` に準拠）。
3. **幅×高さ** と **フィット**（レターボックス / 伸長）を指定。よく使うサイズはプリセットボタンから。
4. **プレビュー＆検証** で警告を確認（色数・サイズ・index0 など）。
5. **BMPに書き出し…** で `Asset/graphics/...` へ保存する。ビルドで `prebuild` に取り込まれます。

UI レイアウトの **sprite** から参照させるには、`Asset/tools/ui_editor/image_manifest.json` に **image_set** を定義します。**GBA UI Editor** のツールバー **「スプライトDB」**（インスペクタの Image Set 横 **「DB…」** からも開けます）で、BMP を選んで番号を振ったりパスを直したりして保存できます（手編集 JSON より楽です）。

## 感情を増やす / 減らす
- 左の「感情の一覧」で、感情カテゴリを **追加・リネーム・削除**、各感情の **数（バリエーション）** を変更できます。
- 変更して変換・ビルドすれば、新しい感情がそのままゲームで使えるようになります（コードの修正は不要）。

## 保存される場所（参考）
- 感情の一覧：`Asset/fixdata/face_catalog.json`
- 画像の割り当て：`Asset/graphics/png/chara_bustup/<プレフィックス>/_assign.json`
- 変換後の素材：`Asset/graphics/sprites/chara/chara/spr_ch_<プレフィックス>_<感情>_<番号>.bmp`

> メモ: イベント等で立ち絵を出すときの `image_id` は `<プレフィックス>_<感情>_<番号>`（例: `mayo_smile_1`）です。
