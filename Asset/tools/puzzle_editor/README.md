# パズルエディタ（Puzzle Editor）

GBA版ソコバンの**ステージ（15×10）**を編集し、ゲーム本番用の `levels.h` を出力するツールです。

---

## 起動方法

1. エクスプローラーでこのフォルダを開く  
   `Asset\tools\puzzle_editor\`
2. **`run_puzzle_editor.bat`** をダブルクリック

**必要環境**

- Windows
- Python 3（`python` コマンドが PATH に入っていること）
- パッケージ: **Pillow**（`pip install Pillow`）  
  - 他のゲームエディタ（UIエディタなど）が動いていれば、通常はそのまま動きます。

起動に失敗したときは、コマンドプロンプトで次を試してください。

```bat
cd プロジェクトのルート\src\gameeditor
python puzzle_editor.py
```

表示されたエラーメッセージをそのまま控えておくと原因を特定しやすいです。

## 画面の見方

```
┌─────────────┬──────────────────────────────┬──────────────┐
│  Levels     │  Layer: BG / FG              │  Level props │
│  (一覧)     │  [15×10 グリッド]            │  Title       │
│             │  [パレット]                  │  Player X/Y  │
│  Add/Dup/Del│                              │  Shady X/Y   │
│             │                              │  Validation  │
├─────────────┴──────────────────────────────┴──────────────┤
│  Reload          Save (JSON + levels.h)                    │
└────────────────────────────────────────────────────────────┘
```

| 領域 | 内容 |
|------|------|
| **Levels** | ステージの一覧。行をクリックすると編集対象が切り替わります。 |
| **グリッド** | マップ本体（横15・縦10マス）。 |
| **Layer** | **Background**＝床ギミック、**Foreground**＝樽・プレイヤーなど。 |
| **Palette** | 現在のレイヤーで使うタイル／オブジェクト。ボタンを押してからグリッドに描きます。 |
| **Level properties** | ステージ名、プレイヤー／シェイディの座標。 |
| **Validation** | 保存前の簡易チェック結果（問題なければ `OK`）。 |

---

## 基本的な編集の流れ

1. 左の一覧で編集したいレベルを選ぶ  
2. **Layer** で **Background** か **Foreground** を選ぶ  
3. **Palette** でタイル／オブジェクトを選ぶ  
4. グリッドを **左クリック**（ドラッグでも連続塗り）  
5. 問題なければ右下 **Save (JSON + levels.h)**  
6. ゲームを **ビルド**（`make` や VS Code のビルドタスク）して実機／エミュで確認

## マウス操作

| 操作 | 動作 |
|------|------|
| **左クリック** | 選んでいるパレットの内容で塗る |
| **左ドラッグ** | 同じ内容で連続塗り |
| **右クリック** | 消す（BG＝床 `FLOOR`、FG＝`NONE`） |

## 「配置できるもの／できないもの」について（重要）

エディタは **2種類のレイヤー** に分かれています。**違うレイヤーだと、どれだけクリックしても置けません。**

| 置きたいもの | 選ぶレイヤー | 例 |
|--------------|--------------|-----|
| 壁・床・スイッチ・氷・矢印・ワープ・穴・出口 | **床（背景）** | 壁、スイッチ、矢印↑ |
| プレイヤー・樽・シェイディ | **前景（キャラ・樽）** | プレイヤー、樽 |

よくある勘違い:

- **前景** のまま壁を置こうとする → 置けない（**床（背景）** に切り替える）
- **床** のままプレイヤーを置こうとする → 置けない（**前景** に切り替える）
- 矢印や氷を置いたのに見えない → **プレイヤーの絵の下に隠れている**ことがある（今の版では床の色も下に描画します）

画面上部の **「配置するもの」プルダウン** で、今何を置くモードか確認できます。

---

## レイヤー別の描き方

### Background（床）

壁・床・スイッチ・氷・矢印・ワープなど、**マップの地形・ギミック**を置きます。

パレットの `数字:名前` は、ゲーム内の床タイルIDです（例: `1:WALL`, `3:SWITCH`, `8:EXIT`）。

### Foreground（前景）

動くもの・押すものを置きます。

| パレット | 意味 |
|----------|------|
| **NONE** | 何もない |
| **PLAYER** | プレイヤー開始位置（**1ステージに1人**） |
| **BARREL** | 樽 |
| **SHADY** | ボス「シェイディ」（いないステージでは置かない） |

**PLAYER** または **SHADY** を置くと、右側の **Player X/Y** または **Shady X/Y** も自動で更新されます。

シェイディを消したいときは、FGで **NONE** を選び、そのマスを右クリックするか塗りつぶしてください。座標は `-1` になります。

---

## 右側パネル（座標・タイトル）

- **Title** … コメント用の名前（`levels.h` の `// --- Level N: ... ---` に出ます）
- **Player X / Y** … プレイヤー開始座標（**0〜14, 0〜9**。左上が (0,0)）
- **Shady X / Y** … シェイディ開始座標。いない場合は **-1 / -1**
- **Apply actors** … 座標を手入力したあと、FGマップ上のマーカーと揃えるときに押す

通常はグリッド上で PLAYER / SHADY を置けば十分です。

## レベル一覧のボタン

| ボタン | 動作 |
|--------|------|
| **Add** | 空に近い新規ステージを追加（外周が壁） |
| **Dup** | 選択中のステージを複製 |
| **Del** | 選択中のステージを削除（**最低1つ**は残します） |

## 保存とデータの行き先

**Save (JSON + levels.h)** を押すと、次の2ファイルが更新されます。

| ファイル | 役割 |
|----------|------|
| `Asset/fixdata/puzzles/levels.json` | **編集用マスター**（人間が読みやすいJSON） |
| `src/game/src/game/levels.h` | **ゲーム本番用**（C++の配列。ビルド時にコンパイルされる） |

**Reload** … ディスク上の内容を読み直します（未保存の変更は確認ダイアログが出ます）。

> エディタは基本的に **JSON を優先**して読み込みます。`levels.json` が無い初回だけ、既存の `levels.h` から取り込みます。

保存後は必ず **プロジェクトをビルド**してください。`levels.h` だけ変えても、ROMに反映されません。

---

## Validation（検証）について

保存前に、次などを簡易チェックします。

- プレイヤーが FG 上で **ちょうど1マス** になっているか
- シェイディが **2体以上** いないか
- 座標が 15×10 の範囲内か

`OK` でも、**クリアできるか・ギミックが意図どおりか**はゲーム内プレイで確認してください（エディタ内シミュレータはありません）。

## 床タイル一覧（Background の数値）

ゲーム仕様（`puzzle_types.h` / 仕様書）に合わせたIDです。

| ID | 名前 | ざっくりした意味 |
|----|------|------------------|
| 0 | FLOOR | 通常の床 |
| 1 | WALL | 壁 |
| 2 | GOAL | ゴールゲート（スイッチ未達時は壁扱い） |
| 3 | SWITCH | 樽を載せるスイッチ |
| 4 | ICE | 滑る床 |
| 5 | CRACKED_1 | ヒビ床（未踏） |
| 6 | CRACKED_2 | ヒビ床（1回踏んだ後） |
| 7 | HOLE | 穴 |
| 8 | EXIT | 出口（クリア用） |
| 9〜12 | ARROW_* | 矢印床 |
| 13 | TOGGLE_SWITCH | 全矢印を回転 |
| 14〜15 | WARP_RED_A/B | 赤ワープペア |
| 16〜17 | WARP_BLUE_A/B | 青ワープペア |

## プレビュー表示について（タイルの絵はどこから？）

グリッドに表示している画像は、**ゲーム本番と同じファイル**です。

| 項目 | 内容 |
|------|------|
| **ファイル** | `Asset/graphics/stills/still_puzzle_map.bmp` |
| **形式** | 256×256、8bitインデックスカラー BMP |
| **切り出し** | 左から 8×8 ピクセル単位（タイル0=床、1=壁、2=プレイヤー…） |

エディタは `renderer.cpp` と同じ考え方で、壁・床・樽・プレイヤーなど **タイルID 0〜8** だけ BMP から表示します。

**矢印・ワープ・氷など** は BMP に専用スロットが無い（または未割当の）ため、エディタでは **色付きの四角** で区別しています。ゲーム内でも同様に床絵の使い回しになっていることがあります。

画面下に「プレビュー画像: still_puzzle_map.bmp」と出ていれば BMP 読み込み成功です。出ていなければパスを確認してください。

## パレットのスクロール

床タイルは **18種類** あるため、パレットは **横スクロール** です。

- パレット下の **スクロールバー** をドラッグ
- パレットの上で **マウスホイール**（横方向）
- **Shift + ホイール** でも横に動きます

各ボタンは **数字 + 日本語名**（例: `3` + `スイッチ`）です。

## トラブルシュート

| 症状 | 対処 |
|------|------|
| `python` が見つからない | Python をインストールし、PATH を通す |
| Pillow エラー | `pip install Pillow` |
| 保存してもゲームが変わらない | **ビルドし直す**（古い ROM を実行していないか確認） |
| ステージがおかしい | `levels.json` を Git で差分確認。必要なら **Reload** 後に再編集 |

## よくある作業例

**新ステージを1つ追加する**

1. **Add** → タイトルを編集  
2. Background で外枠以外を設計  
3. Foreground で樽・PLAYER を配置  
4. Validation が `OK` なら **Save** → ビルド → プレイ  

**既存ステージを少し直す**

1. 一覧でステージを選択  
2. グリッドを修正  
3. **Save** → ビルド
