## 1. 共通ルール（全アセット）

### ファイル形式

| 項目 | 仕様 |
|------|------|
| フォーマット | **BMP（Windows Bitmap）** |
| カラーモード | **8-bit インデックスカラー（パレットモード）** |
| ❌ NG | 24-bit RGB / 32-bit RGBA → ビルドエラーになるので不可 |

### 透過色ルール

**パレットの「インデックス 0 番」の色が透明として扱われます。**

- 透明にしたいピクセルは必ずインデックス 0 にすること
- 透過色として **マゼンタ `#FF00FF` (R:255, G:0, B:255)** を強く推奨
  - 自然画像には出現しない色なので、誤透過が起きにくい

### ⚠️ アンチエイリアス厳禁

キャラクターの輪郭やUIの縁に「半透明のぼかし」をかけないでください。

GBAはアルファブレンドを行わないため、`#FE00FE` のような「ほぼマゼンタ」の
色が透過されず**ピンクのフリンジ（縁取り）**として残ります。

```
OK:  輪郭がジャギー（ギザギザ）でも構いません
NG:  輪郭をぼかして滑らかにする（→ フリンジが出ます）
```

## 2. スプライト（キャラ・アイコン・UIパーツ）

### サイズ規定

GBAで使えるスプライトサイズは決まっています。**以下の一覧から選んでください**。

| 正方形 | 横長 | 縦長 |
|--------|------|------|
| 8 × 8 | 16 × 8 | 8 × 16 |
| 16 × 16 | 32 × 8 | 8 × 32 |
| 32 × 32 | 32 × 16 | 16 × 32 |
| 64 × 64 | 64 × 32 | 32 × 64 |

> **キャラクター立ち絵** は現在 **64 × 64 px** で統一しています。

### カラー数制限

- スプライト 1 枚あたり **最大 16 色**（インデックス 0〜15）
- インデックス 0 は透過色（マゼンタ）なので、実際に使える色は **15 色まで**

### スプライト チェックリスト

- [ ] サイズが上記一覧のいずれかになっている
- [ ] 8-bit インデックスカラー BMP で保存している
- [ ] 透過部分のピクセルがインデックス 0（マゼンタ `#FF00FF`）になっている
- [ ] 輪郭にアンチエイリアスをかけていない
- [ ] 使用色が **15 色以内**（マゼンタ除く）

## 3. 背景CG（イベント背景・スチル画像）

### サイズ規定

**256 × 256 px 固定**

GBAの実描画領域は 240 × 160 px ですが、BG タイルデータとして
256 × 256 px のキャンバスで納品してください。

### カラー数制限（⚠️ 最重要）

背景 CG は、**同時表示される他の背景とパレットバンクを共有します。**

GBA の 4-bit 背景モードは **16 バンク × 16 色 = 合計 256 色** が上限で、
バンク単位での割り当てになります。

#### イベント画面での配分（2枚が同時表示）

| レイヤー | アセット名 | 用途 | 最大カラー数 |
|----------|------------|------|-------------|
| 奥（Priority 3） | `gba_event.bmp` 等 | イベント背景 CG | **240 色（15 バンク）** |
| 手前（Priority 1） | `ui_msg_window.bmp` | メッセージウィンドウ | **16 色（1 バンク）** |

> **超えるとゲームがクラッシュします。**
> 背景 CG 240 色 + UI ウィンドウ 16 色 = 16 バンク（上限ぴったり）

#### 単独表示の背景（ギャラリー・タイトルなど）

他の背景と同時に表示しない画面では **256 色まで**使用できます。

### 背景CG チェックリスト

- [ ] キャンバスサイズが **256 × 256 px**
- [ ] 8-bit インデックスカラー BMP で保存している
- [ ] **イベント画面用**: 使用色が **240 色以内**
- [ ] **単独表示用**: 使用色が **256 色以内**
- [ ] インデックス 0 番の色が絵の中で使われていない
  - インデックス 0 は透過扱いなので、実際の絵の色は 1 番以降に配置

## 4. レイヤー構成の参考（イベント画面）

```
画面手前
  │
  │  Priority 0  テキスト・メッセージ送りアイコン（スプライト）
  │  Priority 1  メッセージウィンドウ枠（背景 BG）※上半分が透明
  │  Priority 2  キャラクター立ち絵（スプライト）
  │  Priority 3  イベント背景 CG（背景 BG）
  │
画面奥
```

メッセージウィンドウ枠（`ui_msg_window`）は**上半分を透明**にすることで、
奥の背景 CG とキャラクターが透けて見える仕組みになっています。

## 5. よくある失敗と対処法

| 症状 | 原因 | 対処 |
|------|------|------|
| キャラ輪郭にピンクの縁が出る | アンチエイリアスで「ほぼマゼンタ」な色が残っている | 輪郭のぼかしをなくす。透過色は完全な `#FF00FF` のみ |
| 背景がガビガビ・色化けする | 2 枚の背景のパレットが衝突している | カラー数を制限する（イベント CG は 240 色以内） |
| 起動直後にゲームがクラッシュする | パレットバンクが枯渇している | 背景 CG を 240 色以内に収める |
| 上半分が白/黒/変な色で表示される | インデックス 0 に実際の色が割り当たっている | インデックス 0 を空け、マゼンタを 0 番に固定する |

## 6. 推奨ソフトウェアと設定

### Aseprite（推奨）

1. カラーモード → **Indexed**
2. `Sprite > Properties > Transparent Color` を **0** に設定
3. パレットの **スロット 0 をマゼンタ `(255, 0, 255)`** に固定
4. 書き出し → **BMP** 形式で保存

### GIMP

1. `画像 > モード > インデックス` で変換
2. 最大色数を用途に合わせて指定（スプライト: 16、背景: 240 or 256）
3. エクスポート → **Windows BMP** 形式で保存

### Photoshop

1. `名前をつけて保存 > BMP`
2. カラー深度 → **8 bit**
