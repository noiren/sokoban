# GBA Sokoban - タイル・オブジェクト対応表 (Tile & Object Reference)

このドキュメントは、ゲーム内で使用される各種床タイル (`BgTile`) および前景オブジェクト (`FgObj`) と、実際にゲーム画面（GBA）上で描画されるタイルシートのインデックス（Tile ID）との対応表です。

---

## 1. 前景オブジェクト (FgObj)
キャラクターや動かせるものなどのレイヤーです。
マップ座標に `FgObj` が存在する場合、床タイルの上から優先的に描画されます。

| FgObj Enum 定義 | 整数値 | 条件・床タイルの状態 | 描画 Tile ID | 画面上での見た目・説明 |
|:---|:---:|:---|:---:|:---|
| `FgObj::NONE` | `0` | なし | - | 何も描画しない（床タイルがそのまま見えます） |
| `FgObj::PLAYER` | `1` | 制限なし | **2** | **プレイヤー（主人公：黒い◆）** |
| `FgObj::BARREL` | `2` | 床が `BgTile::SWITCH` | **5** | **樽＋スイッチ（茶色の樽、下に緑のスイッチ）** |
| `FgObj::BARREL` | `2` | 上記以外 | **3** | **樽（茶色の四角）** |
| `FgObj::SHADY` | `3` | 制限なし | **6** | **シェイディ（NPC/敵キャラ：紫色の四角）** |

---

## 2. 床タイル (BgTile)
マップのベースとなる床・ギミックのレイヤーです。前景オブジェクトがない場合、このタイルが描画されます。

| BgTile Enum 定義 | 整数値 | 描画 Tile ID | 画面上での見た目・説明 | 物理的な挙動・ゲームロジック |
|:---|:---:|:---:|:---|:---|
| `BgTile::FLOOR` | `0` | **0** | **床（薄灰色タイル）** | 通常の通行可能なプレーンな床 |
| `BgTile::WALL` | `1` | **1** | **壁（額縁・枠線ブロック）** | ┏ ┗ などの角や枠線で構成された立体的な壁 |
| `BgTile::GOAL` | `2` | **22** / **21** | **ゴールゲート（開門 / 閉門）** | スイッチ作動で **22** (開門) になり通行可能になります。それ以外は **21** (閉じた鉄格子門) です。 |
| `BgTile::SWITCH` | `3` | **4** | **スイッチ（緑＋赤ボタン）** | 緑の土台の真ん中に一回り小さい赤いスイッチボタンがあります。 |
| `BgTile::ICE` | `4` | **9** | **氷床（ツルツルした氷）** | 光り輝く凍った水色。乗ると滑り続けます。 |
| `BgTile::CRACKED_1` | `5` | **10** | **ヒビ床・未踏（ひび床）** | じゃっかんひび割れている床。一度乗ると `CRACKED_2` に変化します。 |
| `BgTile::CRACKED_2` | `6` | **11** | **ヒビ床・ひび入り（ボロボロ）** | 崩落寸前の床。再度乗ると崩れて `HOLE`（穴）になります。 |
| `BgTile::HOLE` | `7` | **7** | **穴（ヒビ枠＋黒穴）** | 外側にヒビが微妙に残っていますが真ん中は真っ黒な深い穴です。 |
| `BgTile::EXIT` | `8` | **8** | **出口（脱出ポータル）** | ゴール(門)とは異なる、脱出用の光り輝く星/ワープゾーン。 |
| `BgTile::ARROW_UP` | `9` | **12** | **上矢印床（白背景＋青矢印）** | 白いタイルの上に上向きの青い矢印が描かれています。 |
| `BgTile::ARROW_DOWN` | `10` | **13** | **下矢印床（白背景＋青矢印）** | 白いタイルの上に下向きの青い矢印が描かれています。 |
| `BgTile::ARROW_LEFT` | `11` | **14** | **左矢印床（白背景＋青矢印）** | 白いタイルの上に左向きの青い矢印が描かれています。 |
| `BgTile::ARROW_RIGHT`| `12` | **15** | **右矢印床（白背景＋青矢印）** | 白いタイルの上に右向きの青い矢印が描かれています。 |
| `BgTile::TOGGLE_SWITCH`| `13` | **16**| **トグルスイッチ（回転渦巻）**| 床の上に、回転を表現するくるくるのオレンジ渦巻が乗っています。 |
| `BgTile::WARP_RED_A` | `14` | **17** | **赤ワープA（赤ランプ丸タイル）**| 丸いタイルの上に赤いランプが乗っているワープゾーンです。 |
| `BgTile::WARP_RED_B` | `15` | **18** | **赤ワープB（赤ランプ丸タイル）**| 丸いタイルの上に赤いランプが乗っているワープゾーンです。 |
| `BgTile::WARP_BLUE_A`| `16` | **19** | **青ワープA（青ランプ丸タイル）**| 丸いタイルの上に青いランプが乗っているワープゾーンです。 |
| `BgTile::WARP_BLUE_B`| `17` | **20** | **青ワープB（青ランプ丸タイル）**| 丸いタイルの上に青いランプが乗っているワープゾーンです。 |
| `BgTile::EMPTY` | `17` | **17** | **マップ外（透明）** | マップの有効範囲外の描画用タイル。 |

> [!NOTE]
> - 「※仮描画」と記載しているものは、現状アセット内に専用のグラフィックがなく、床（Tile ID `0`）やスイッチ（Tile ID `4`）のグラフィックを使いまわして表示されています。
> - 今後グラフィックを追加する場合は、`make_puzzle_bg.py` で Tile ID `8`〜`16` などの空きスペースに専用のグラフィックを描画し、`renderer.cpp` の描画ロジックで該当する Tile ID に振り分けることで表示を差し替えることができます。

## 3. レンダラーでのタイル処理の仕組み (`renderer.cpp` より)
ゲームエンジン（`PuzzleEngine`）が管理するマップデータは `bg_map`（床）と `fg_map`（前景オブジェクト）の2つの二次元配列で構成されています。
`renderer.cpp` 内の `render_draw_map` 関数において、以下のように2つの配列を重ね合わせて、最終的にGBAに表示する1枚のタイルマップへ変換しています。

```cpp
// bg_map と fg_map を合成して描画用タイルを決定する
if (fg == FgObj::PLAYER) {
    render_tile = 2; // プレイヤー
} else if (fg == FgObj::BARREL) {
    render_tile = (bg == BgTile::SWITCH) ? 5 : 3; // 樽（スイッチ上なら5、それ以外なら3）
} else if (fg == FgObj::SHADY) {
    render_tile = 6; // シェイディ
} else {
    // 前景オブジェクトがない場合は、床タイルの種類に応じて描画
    switch (bg) {
        case BgTile::WALL:          render_tile = 1; break;
        case BgTile::GOAL:          render_tile = switches_active ? 4 : 1; break;
        case BgTile::SWITCH:        render_tile = 4; break;
        case BgTile::HOLE:          render_tile = 7; break;
        case BgTile::EXIT:          render_tile = 8; break;
        case BgTile::CRACKED_2:     render_tile = 4; break;
        case BgTile::TOGGLE_SWITCH: render_tile = 4; break;
        default:                    render_tile = 0; break; // FLOORやICE, 矢印, ワープなど
    }
}
```
